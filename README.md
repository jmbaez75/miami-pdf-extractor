🌴 Miami PDF Extractor (M.P.E)

The ultimate automated solution for batch PDF data extraction.
📖 The Origin Story

The Miami PDF Extractor (M.P.E) was built because the bureaucratic process of justifying public funds in Spain is a disaster.

As a professional in R&D, I spent too much time manually transcribing data from payrolls, bank statements, and invoices into spreadsheets to satisfy audit requirements. These documents are rigid and repetitive, but since the administration doesn't provide tools to process them, we were forced to do the grunt work by hand. It’s inefficient, it’s frustrating, and it’s a waste of time.

I developed M.P.E as a direct response to this. It is a tool designed to scrape data from those documents automatically. It leverages the fixed structure of these PDFs to extract the necessary information without the need for manual data entry.

I named it M.P.E because, while I was buried in that pile of mandatory paperwork, all I could think about was being anywhere else, doing nothing. It’s a tool for people who want to finish the audit process as fast as possible so they can move on to actual work—or take a break.
🚀 Key Features

    Batch Processing: Process hundreds of PDFs in seconds.

    Smart Mapping: Dynamically map PDF layouts without complex coding.

    Automated Cleaning: Built-in filter system to sanitize and format data on the fly.

    Error Resilience: Real-time feedback and detailed error handling to ensure data integrity.

    Responsive Web Interface: A sleek, intuitive dashboard to manage your workflows.

🛠 Tech Stack

    Backend: Python (Django)

    Frontend: HTML5, CSS3, JavaScript

    Data Handling: JSON, CSV, Pandas

📋 How it Works

    Map Execution: Scan a sample PDF to generate your coordinate reference map.

    Output Columns: Easily configure which data fields you want to extract.

    Filters: Apply rules to clean your text (e.g., currency formatting, removing symbols).

    Execute Batch: Let the tool do the heavy lifting while you monitor the progress.
